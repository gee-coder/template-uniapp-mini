# 支持说明

## 1. 仓库定位

本仓库是创业技术底座中的 UniApp 小程序模板，默认提供登录页、首页、用户中心与统一 JWT 请求封装能力。

该仓库提供模板级支持，适合快速搭建多端小程序基座，不承诺生产 SLA。

## 2. 提问前建议先检查

- [README.md](./README.md)
- [CONTRIBUTING.md](./CONTRIBUTING.md)
- [SECURITY.md](./SECURITY.md)
- [RELEASE.md](./RELEASE.md)
- `pages.json`、`manifest.config.ts`、`src/api`、`src/common`

## 3. 适合提交公开 Issue 的场景

- 登录、鉴权守卫、Token 持久化或页面跳转存在可复现问题
- 文档、平台构建说明或接口约定与实际行为不一致
- 可沉淀为多端模板能力的增强建议
- UniApp、Vite 或依赖升级后出现兼容性回归

## 4. 不建议提交公开 Issue 的场景

- 平台密钥、敏感配置、生产接口、真实用户数据泄露
- 小程序审核材料、商务资源或单项目品牌定制需求
- 涉及安全绕过、越权路径或敏感网络信息的问题

安全相关内容请改走 [SECURITY.md](./SECURITY.md)。

## 5. 提问时建议包含的信息

- Node.js 与 npm 版本
- 目标平台：微信小程序或抖音小程序
- 最短复现步骤
- 编译日志、运行截图、接口返回与控制台报错
- 是否对模板默认登录 / 鉴权流程做过二次改造

## 6. 响应优先级

- 影响登录与首页主流程的可复现问题优先
- 构建、类型检查与平台兼容性问题优先
- 通用能力增强会结合整体模板方向评估

## 7. Fork 使用建议

如果你基于本模板进行业务开发，建议同步补齐团队自己的平台账号配置流程、审核发布流程与支持渠道。
