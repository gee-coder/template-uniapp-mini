# 提交规范

建议采用 Conventional Commits 风格。

## 1. 推荐格式

```text
type(scope): subject
```

## 2. 常用类型

- `feat`
- `fix`
- `docs`
- `refactor`
- `test`
- `chore`
- `build`
- `ci`

## 3. 示例

```text
feat(login): add account password sign-in page
fix(build): align uniapp compatibility layer
docs(readme): add publish workflow
refactor(api): simplify token refresh request
```
